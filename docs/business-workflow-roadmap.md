# 业务流程改造路线图（确认稿 v2）

## 1. 需求总览（已合并你这几轮补充）

1. 新客线索成交后，立即进入收费录入。
2. 老客二次开发不是“手工选客户”，而是“搜索客户名或老板/联系人”定位后再成交录费。
3. 同一客户可并行多条收费项目（注册、代账、退税、咨询、课程、海外注册等）。
4. 收费支持按期/按次；收款方式只保留预收/后收。
5. 按期费用需填写开始月份，默认结束月份 = 开始月份 + 11（可改）。
6. 金额口径要明确“月费/年费/单次/周期总价”含义，避免歧义。
7. 续费时复制上期条目并默认顺延 +1 年，可手动调整。
8. 支持提前结束合同，并冲减后续应收。
9. 一笔实收可冲抵多个项目：系统给默认抵扣顺序，同时支持手动改分配。
10. 需要客户明细账：客户检索 + 时间段筛选，输出摘要/时间/应收/实收/余额（可正可负）。

## 2. 分步实施（每步可验收）

### 步骤 1（已完成）：收费模型基础化

- 后端 `billing_records` 增加收费业务字段与兼容迁移。
- 收费录入三入口统一字段（收费页、客户页、线索转化页）。
- 按期/按次默认规则落地（按期默认 +11 月，按次默认当天到期）。

### 步骤 2（已完成）：老客二次开发闭环（最小可用）

- `客户开发`页支持按客户名/联系人搜索，创建“老客二开线索”。
- 老客二开线索成交时复用原客户，不重复创建客户档案。
- 成交后直接进入录费流程，形成“开发 -> 成交 -> 收费”闭环。

### 步骤 3（已完成）：订单分派与可见性

- 给收费项目增加执行分派（如注册专员/交付人员）。
- 可见性规则：会计 + 被分派执行人 + 老板/管理员可见费用标准与进度。

### 步骤 4（已完成）：收款分摊（自动建议 + 手工调整）

- 实收录入时自动生成抵扣建议（按优先顺序）。
- 允许手工改分摊并保存。
- 台账联动重算每个项目的已收/欠款。

### 步骤 5（已完成）：续费与提前终止

- 系统生成到期提醒，业务负责人确认是否续费。
- 一键续费复制上期记录，默认时间顺延一年。
- 提前结束合同时按规则冲减未发生期间应收。

### 步骤 6（已完成）：客户明细账

- 按客户 + 起止年月查询。
- 字段：摘要、时间、应收、实收、余额（正/负）。
- 提供月度汇总 + 明细下钻，支持会计、相关执行人、管理员查看。
