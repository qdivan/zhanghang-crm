# 代账客户开发与收款系统（MVP）

## 目标
把当前 3 份 Excel（客户开发、客户维护、收费记录）整合为一个系统，支持多人协作、流程闭环和手机端使用，降低会计学习成本。

## 核心范围（仅保留现有业务必需）
1. 客户开发：线索录入、跟进记录、状态流转（未开发/跟进中/转化成功/失败）、提醒。
2. 客户分配：开发成功后分配给会计；转化后进入客户列表并生成独立客户档案页；会计账号先走本地账号，后续接入群晖 LDAP。
3. 收费管理：支持月付/年付、一次性任务收费、预收款、欠款结清、有效期。
4. 催款：会计和老板都可查看应收并发起催款记录。
5. 收款录入：收款可关联固定周期服务或一次性任务，形成账款流水。
6. 成本录入：按客户记录成本项；仅老板和管理员可查看统计。
7. 权限角色：老板、会计、管理员。
8. 界面要求：尽量沿用 Excel 布局和字段命名，降低迁移学习成本。
9. 明细要求：支持“总览 -> 单条明细页”操作方式，贴近 Excel 每个客户一个 sheet 的习惯。
10. 用户管理：管理员面板仅对有权限角色显示；管理员可管理全部用户，老板可管理除管理员外的用户。
11. 管理员可配置 LDAP 并同步群晖 LDAP 账号到系统用户，支持查看操作日志与最近登录时间。
12. 客户开发首页只保留核心字段，线索与客户差异信息在“转化”步骤填写。

## 非目标（MVP 不做）
1. 通用 ERP 能力（采购、库存、生产、复杂财务总账等）。
2. 复杂审批流、复杂多组织结构。
3. LDAP 单点登录（先不做，当前仅提供 LDAP 设置与账号同步）。

## 关键约束
1. 页面要简洁、少菜单，优先“表格式”操作体验。
2. 前后端完整，可在手机上可用（优先响应式 Web/PWA）。
3. Excel 全功能可覆盖，支持历史数据导入。

## 本轮新增约束（按 Excel 复盘）
1. 必须同时支持两套客户开发字段模板：
   - `客户跟进表` 模板（国家、企业性质、服务方式、收费标准等）
   - `转化2026` 模板（地区、联络开始时间、备用2/3/4 等）
2. 收费台账以 `周 (2)` 为主，保留 `总费用/月费用/代账周期/到期日/付款方式/扩展字段`。
3. 颜色状态不再只靠单元格色块，系统内必须落显式状态字段。
4. `备用字段` 与自由文本字段不能被简化删除（历史已在使用）。
