# Requirements - 代账客户开发与收款系统 MVP（Excel 全字段覆盖版）

## Problem Summary
当前业务由 3 份 Excel 驱动，主要问题不是“缺系统”，而是“字段与流程没有统一数据底座”。本期目标是：按 Excel 真实字段和操作习惯重建系统，保证原表功能可迁移、可协作、可追溯。

## Source Of Truth
1. 字段与逻辑明细见：`excel-analysis.md`。
2. 实施时以 `周 (2)` 作为收费主台账模型，`周` 作为兼容导入模型。
3. 两套客户开发模板必须并存（跟进模板 + 转化模板）。

## Acceptance Criteria

### A. 账号与权限
- [ ] `OWNER`、`ADMIN`、`ACCOUNTANT` 本地账号可登录。
- [ ] `ACCOUNTANT` 仅可见自己负责的客户与收费记录。
- [ ] `OWNER`、`ADMIN` 可全量查看并维护收费台账与扩展备注。
- [ ] 核心操作（创建线索、跟进、转化、收费台账新增/修改）记录审计信息。
- [ ] 提供“管理员面板”入口，且仅 `OWNER`、`ADMIN` 可见。
- [ ] `ADMIN` 可管理全部本地用户（含管理员）。
- [ ] `OWNER` 可管理除 `ADMIN` 外的用户（不可创建/修改管理员账号）。
- [ ] 用户列表显示最近登录时间（`last_login_at`）。
- [ ] 管理员面板支持删除本地账号（仅无关联业务数据账号可删，且不可删当前登录账号）。
- [ ] 操作日志中的用户更新需记录变更字段 diff（如 role/is_active/password）。

### B. 客户开发字段覆盖（两套模板）

#### B1 跟进模板（对应 `客户跟进表`）
- [ ] 支持字段：
  - [ ] 序号、等级、客户号+公司名、国家、服务开始时间、企业性质、服务方式
  - [ ] 对接人及电话、微信、其他联系人、主营产品、介绍
  - [ ] 收费标准、首期账单期间、最后跟进日期、提醒值
- [ ] 支持明细跟进表字段：跟进日期、跟进反馈、备注。

#### B2 转化模板（对应 `转化2026`）
- [ ] 支持字段：
  - [ ] 序号、等级、公司名、地区、联络开始时间、联系人（微信号）
  - [ ] 主营、电话、传真、其他联系方式、备注、介绍
  - [ ] 备用2、备用3、备用4、最后跟进日期、提醒值
- [ ] 支持地址资源字段：分类、联系方式、资源说明、后续动作。

#### B3 通用流程
- [ ] 线索状态支持：`NEW`、`FOLLOWING`、`CONVERTED`、`LOST`。
- [ ] 转化成功后可创建客户并分配会计。
- [ ] 转化动作必须指定会计（仅允许 `ACCOUNTANT` 角色），不允许分配给老板/管理员。
- [ ] 转化弹窗可填写“转化后客户名称/联系人/电话”，用于处理线索与客户档案字段差异。
- [ ] `序号` 与“客户明细”视图一一对应（总览 -> 明细）。
- [ ] 支持维护完整联系人字段（联系人、电话、微信、传真、其他联系方式）。
- [ ] 支持维护“最后跟进日期”与“提醒值”，并可由跟进记录自动回写。
- [ ] 线索转化后，前端可直接进入对应客户页面（避免“只变灰”但无去向）。
- [ ] 客户开发列表中每条记录都可进入单独详情页（对齐 Excel 的“总览 -> 明细页”）。
- [ ] 支持转化撤销（误操作可回退），但若已有关联收费记录则禁止撤销并给出原因。

#### B5 客户列表与客户档案
- [ ] 新增客户列表页面（来源于已转化线索）。
- [ ] 客户列表支持关键词检索、角色权限过滤（会计仅看自己客户）。
- [ ] 每个客户有独立档案页，展示：
  - [ ] 客户基础信息（名称、联系人、电话、分配会计）
  - [ ] 来源线索完整字段（贴近 Excel 明细上半区）
  - [ ] 跟进记录时间线（贴近 Excel 明细下半区）
- [ ] 客户档案页可继续录入跟进记录（与线索跟进数据一致）。
- [ ] 已分配会计可在客户档案录入跟进（即使线索 owner 不是该会计）。
- [ ] 客户档案页可编辑客户基础信息与来源线索关键信息。

#### B4 地址资源（`转化2026`）
- [ ] 支持地址资源记录：`分类/区域`、`联系方式`、`资源说明`、`后续动作`。
- [ ] 地址资源支持新增、编辑、列表检索。
- [ ] 地址资源可作为新建线索时的来源参考。

### C. 收费台账覆盖（对应 `周 (2)`）
- [ ] 支持字段：
  - [ ] 序号（数字）、客户ID、名称、总费用、月费用
  - [ ] 代账周期、到期日、付款方式、备注说明、扩展字段
  - [ ] 台账状态：`CLEARED(清账)`、`FULL_ARREARS(全欠)`、`PARTIAL(部分收费)`
- [ ] 收费记录必须可关联到客户档案（`customer_id`），保证“收费客户=客户列表客户”。
- [ ] 收费记录可显示对应会计（由客户分配会计继承），不再用 `周1/周20` 表示会计归属。
- [ ] 付款方式支持并可扩展：`预收`、`年底收`、`半年付费`、`后收`、`核定不收费`。
- [ ] 允许“收款日期+服务期间”文本化保存（兼容历史表达）。
- [ ] 收费记录支持显式状态字段，替代仅靠颜色判断。

### D. 催收与收款语义
- [ ] 支持记录欠费、部分收费、清账状态（显式字段）。
- [ ] 支持备注中附加费用语义：退税、年报、账本、审计、注销、稳岗补贴。
- [ ] 支持一次性事项并入收费记录（不强制拆分为单独模块）。
- [ ] 支持催收日志（催收人、催收时间、沟通内容、下次跟进日期）。
- [ ] 支持收款日志（收款时间、金额、收款类型、是否预收、是否结清、备注）。
- [ ] 收款日志可关联台账记录，并累积得到已收金额/未收金额。
- [ ] 催收日志金额固定为 `0`，只有收款日志允许填写金额。

### E. 界面与可用性
- [ ] 页面以“Excel 风格宽表”为主（字段顺序尽量与原表一致）。
- [ ] 客户开发总览首页只展示核心字段（名称、联系人、电话、来源、状态、提醒、最后反馈）；完整字段进入详情页。
- [ ] 客户开发总览默认排序中，“已转化”记录置底，不与待开发线索混排在顶部。
- [ ] 保留中文字段名展示，降低迁移学习成本。
- [ ] 手机端可查看和维护核心字段（线索、跟进、收费记录）。
- [ ] 手机端主导航需为抽屉式菜单，避免侧栏挤压主内容。
- [ ] 客户开发页提供“跟进记录历史”查看，替代 Excel 明细页下半区。
- [ ] 客户开发页提供“详情”入口，直接查看单条线索的完整档案。
- [ ] 提供“客户列表/客户档案”入口，形成开发与客户管理的连续路径。
- [ ] 客户开发页公司名支持点击进入对应详情（已转化进客户档案，未转化进线索详情）。
- [ ] 收费页提供“催收/收款记录”查看与录入。
- [ ] 所有日期字段支持两种输入方式：日历弹出选择 + 键盘按 `YYYYMMDD` 或 `YYMMDD` 直接录入。
- [ ] 管理员面板支持：用户列表检索、创建用户、编辑角色/启停状态、重置密码（本地账号）。
- [ ] 管理员面板错误提示统一中文，避免中英文混用。
- [ ] 管理员面板支持操作日志查看（按动作/对象/关键词筛选）。
- [ ] 管理员面板支持 LDAP 设置与手动同步账号（群晖 LDAP 场景）。
- [ ] 客户开发页提供“流程说明”按钮，说明线索/跟进/转化与状态含义，以及“模板筛选”的用途。

### F. 导入迁移
- [ ] 支持 3 文件导入预览（识别主表、明细、收费表）。
- [ ] 导入结果可输出：成功条数、失败条数、失败原因。
- [ ] 导入后可生成核对报告（客户数、跟进数、收费记录数）。

## Constraints & Dependencies
1. 首版支持本地账号 + LDAP 手动同步；LDAP 单点登录（二期）。
2. 不引入通用 ERP 流程，优先业务贴合度。
3. 颜色语义必须转换为结构化状态字段，避免单靠 UI 色块。
