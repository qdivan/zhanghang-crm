# 产品背景（代账客户开发与收款系统）

## 业务目标
用一个轻量系统替代当前 3 份 Excel 的人工同步，覆盖客户开发、客户维护、收费收款、催款、成本记录全流程。

## 当前痛点
1. 多个 Excel 分散维护，数据不同步，跨人协作容易冲突。
2. 会计视角和老板视角分离，难以实时看到应收、欠款、成本与利润。
3. 跟进提醒、催款记录和一次性任务收费缺乏统一台账。
4. 学习成本要求高：新系统必须尽量继承 Excel 的表格布局和字段命名。

## 关键用户与职责
1. 老板：查看全局客户、应收、欠款、成本、利润；参与催款。
2. 会计：维护自己负责客户、查看应收任务、登记收款、催款。
3. 管理员：账号管理、客户分配、系统配置（后续 LDAP 接入）。

## 核心流程
1. 线索录入 -> 跟进 -> 转化成功/失败。
2. 转化成功客户分配给会计（首版本地账号，后续 LDAP）。
3. 为客户配置收费项（按月/按年/一次性任务）和有效期。
4. 系统生成应收账单与催款任务，会计和老板可催款并记录结果。
5. 收款登记（预付、当期结清、历史欠款结清），自动更新账龄与状态。
6. 按客户录入成本项，老板/管理员查看利润与统计报表。

## 成功标准（MVP）
1. 3 份 Excel 的字段与流程可在系统中完整操作。
2. 任一客户可在 3 分钟内完成“创建 -> 分配 -> 建收费项 -> 录入收款”闭环。
3. 会计 1 天内完成上手（不依赖 ERP 训练）。
4. 手机端可完成日常查看、催款、收款登记。

## 明确不做（MVP）
1. 通用 ERP 模块（采购、库存、生产、复杂总账）。
2. 复杂审批工作流与多法人体系。
3. 首版 LDAP 打通（只预留接口）。
