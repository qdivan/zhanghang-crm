# Dev 试运行每日清单

## 目标
- 让老板和会计每天至少完成一轮真实使用。
- 不靠主观印象，直接把问题记到 `/Users/shangyifan/Documents/New project/output/test-reports/dev-trial-feedback-log.md`。

## 使用规则
1. 每天先打开：
- Web：`http://127.0.0.1:32080`
- 反馈日志：`/Users/shangyifan/Documents/New project/output/test-reports/dev-trial-feedback-log.md`
2. 发现问题时，不要先在聊天里描述。
3. 先写日志，再决定是否需要立即修。
4. 如果出现 `P0/P1`，当天停止新增需求，先修缺陷。

## 老板每日清单
1. 登录系统。
2. 查看工作台待办是否正常。
3. 打开 `客户开发`：
- 查看今日需要跟进的线索。
- 新增或编辑至少 1 条线索。
4. 打开 1 个 `客户档案`：
- 查看客户时间线。
- 新增 1 条客户事项，或核对已有事项状态。
5. 打开 `收费明细`：
- 查看应收、逾期、未收是否正常。
- 点击 1 家公司，打开往来账。
6. 打开 `到账核对`：
- 切换 1 个入账账户。
- 核对流水是否能正常显示。
7. 打开 `常用资料` 或 `公开资料`：
- 核对内容可查、可筛选。

## 会计每日清单
1. 登录系统。
2. 查看 `收费明细`：
- 检查逾期和待催收项目。
- 打开 1 家公司的往来账。
3. 打开 1 个 `客户档案`：
- 查看客户时间线和提醒事项。
- 补 1 条已办事项或结果记录。
4. 如果有真实收款：
- 录入收款并选择入账账户。
5. 查看 `常用资料`：
- 搜 1 条催单或培训模板。
6. 如有挂靠地址业务：
- 打开 `挂靠地址`，核对可查。
